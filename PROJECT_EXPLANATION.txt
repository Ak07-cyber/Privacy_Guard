================================================================================
                    PASSIVEGUARD - PROJECT EXPLANATION
              ML-Based Passive Bot Detection (CAPTCHA Replacement)
================================================================================

PROJECT OVERVIEW
--------------------------------------------------------------------------------
PassiveGuard is a machine learning-powered bot detection system that replaces
traditional CAPTCHAs. Instead of asking users to solve puzzles or click images,
it passively analyzes browser environment and user behavior to determine if a
visitor is human or bot - completely invisible to the user.

Problem Statement: MeitY Problem ID 1672 - UIDAI Portal Bot Detection
Target: Government portals requiring secure, accessible bot detection

           ▼

================================================================================
                    OPTIMIZATION FOR LOW-END DEVICES
================================================================================

PassiveGuard is specifically designed to work on low-end devices with minimal
resource usage:

1. LIGHTWEIGHT SDK (< 15KB GZIPPED)
───────────────────────────────────
   • No heavy dependencies
   • Tree-shakeable modules
   • Lazy loading of collectors
   • Only essential code included

2. MINIMAL CPU USAGE
────────────────────
   • Event listeners use passive mode
   • Throttled data collection (100ms intervals)
   • No continuous polling
   • Efficient data structures (no memory leaks)

   // Example: Throttled mouse tracking
   mousemove event → throttle(100ms) → store data
   
   Instead of capturing every pixel movement (1000+ events/second),
   we sample at 10 events/second maximum.

3. LOW MEMORY FOOTPRINT
───────────────────────
   • Maximum 1000 events stored in memory
   • Circular buffer discards old events
   • No large data structures
   • Automatic cleanup on destroy()

   Memory Usage Comparison:
   ┌────────────────────────┬─────────────┐
   │ Traditional CAPTCHA    │ ~500KB      │
   │ reCAPTCHA v3           │ ~300KB      │
   │ PassiveGuard SDK       │ ~50KB       │
   └────────────────────────┴─────────────┘

4. BATTERY EFFICIENT
────────────────────
   • No WebGL/Canvas operations after initial fingerprint
   • No continuous animations
   • No background workers unless necessary
   • Event listeners removed on destroy()

5. NETWORK EFFICIENT
────────────────────
   • Single API call for verification
   • Compressed JSON payload (< 5KB)
   • No polling or WebSocket connections
   • Supports offline detection

6. WORKS ON OLD BROWSERS
────────────────────────
   • Compiled to ES2015 (IE11+ compatible)
   • Graceful degradation for missing APIs
   • Polyfills not required for core functionality

   Feature Detection Example:
   if (navigator.deviceMemory) {
       // Use device memory
   } else {
       // Use default value
   }

FULL INTEGRATION EXAMPLE
────────────────────────

<!DOCTYPE html>
<html>
<head>
    <script src="passiveguard.min.js"></script>
</head>
<body>
    <form id="loginForm">
        <input type="email" name="email" required>
        <input type="password" name="password" required>
        <button type="submit">Login</button>
    </form>

    <script>
        // Initialize on page load
        const guard = new PassiveGuard({
            apiEndpoint: 'http://localhost:8000',
            siteKey: 'my-website'
        });
        
        guard.init();

        // Verify on form submit
        document.getElementById('loginForm').onsubmit = async (e) => {
            e.preventDefault();
            
            const result = await guard.verify();
            
            if (result.isHuman) {
                // Human verified - submit form
                e.target.submit();
            } else {
                alert('Verification failed');
            }
        };
    </script>
</body>
</html>
